# Personal Knowledge Assistant (Telegram Bot to Cloudflare Worker)

This project creates a personal knowledge assistant accessible via a Telegram Bot. It captures text, web links, and images, converts them to Markdown, auto-categorizes them, and answers questions based on the accumulated knowledge base.

## User Review Required

> [!WARNING]
> **Constraint with Microsoft's `markitdown` Library on Cloudflare Workers**
> You specified deploying on **Cloudflare Workers** and using Microsoft's **`markitdown`** library. `markitdown` is a Python-based library that relies on underlying native modules (like PDF parsers). Cloudflare Workers support Python via Pyodide (WebAssembly), which often struggles or flat-out fails to run native Python dependencies that don't have pre-built WASM wheels. 
> 
> **Proposed Alternatives:**
> 1. **Pure TypeScript/JS Cloudflare Worker:** Build equivalent ingestion flows using JS libraries (e.g., `cheerio` + `turndown` for web links, `pdf.js` for PDFs, Cloudflare AI for image parsing). This ensures seamless deployment on Cloudflare natively.
> 2. **Hybrid Approach:** Deploy the main bot + storage on Cloudflare Worker (JS/TS), and host a tiny Python Python API (e.g., on Render or Fly.io) specifically wrapping `markitdown` to convert files on the fly. 
> 3. **Try Pyodide on CF:** We can attempt building a Python Cloudflare Worker first, though if `markitdown` compilation fails due to missing Pyodide wheels, we will have to pivot.
> 
> *Please let me know which approach you prefer in your feedback! My recommendation is Option 1 since it utilizes native JS tooling making it resilient and fast within the Cloudflare edge environment.*

## Proposed Changes

### 1. Cloudflare Storage Setup (R2 & KV/D1)
- Provision **Cloudflare R2 Bucket**: Best for storing `.md` files and generated `.html` answers persistently.
- Provision **Cloudflare KV or D1**: Keep track of user IDs, categories, file metadata, and secure access tokens to view answers.

---

### 2. Telegram Bot Webhook Integration (Cloudflare Worker)
- Scaffold a fresh TypeScript Cloudflare Worker (suggest using `Hono` router for clean endpoint management).
- Build the `/webhook` endpoint to consume updates from the Telegram Bot API.
- Safely route incoming media (text, photos) to the Parser block.
- Handle specific commands: `/ask [question]` to trigger the Q&A pipeline.

---

### 3. Markdown Parser & Auto-Categorization block
- Implement media fetching from the Telegram API servers.
- **Conversion Phase:** Apply the chosen Markdown strategy (e.g., parsing web links, converting images to text descriptions via LLM).
- **Categorization Phase:** Prompt an LLM via API (e.g., "Analyze the following markdown text and return a 1-word category/topic").
- Store the formatted, categorized `.md` payload into the R2 bucket.

---

### 4. Multi-LLM Q&A Pipeline Engine
- Retrieve existing `.md` files from Cloudflare Storage. (For a growing knowledge base, we may eventually need to use Cloudflare Vectorize for semantic chunk search rather than loading *all* files at once, but we will start with a grouped injection).
- Implement a pluggable AI client interface utilizing native JS `fetch` to interface seamlessly with:
  - DeepSeek API
  - Google Gemini API
  - OpenAI ChatGPT API
  - Anthropic Claude API
- Synthesize an answer to the user's prompt by piping the accumulated documents into the chosen provider.

---

### 5. Private HTML Answer Generation
- Once an answer is generated by the LLM, format it natively as Markdown and persist it to `answers/<uuid>.md` in R2.
- Build an endpoint `/answer/<uuid>` that dynamically converts the Markdown to an HTML string (using `marked` or similar) with beautiful modern styling (Tailwind/CSS).
- Secure the endpoint via an authentication parameter (e.g., `?token=<secure_token>` tied to the user's Telegram ID).
